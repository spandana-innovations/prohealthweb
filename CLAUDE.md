# ProHealth Home Care — project guide for Claude Code

This file is context for Claude Code. Read it before making changes.

## What this is

The marketing website + backend for ProHealth Home Care, Inc., a California home
health / hospice agency (Sacramento HQ, six offices, 20 counties).

- **Site**: a static site generated by Python into `./site`, deployed to Cloudflare Pages.
- **Backend**: a Cloudflare Worker in `./worker` — lead capture, job applications,
  CCPA data requests, and an admin dashboard.

## The one rule that matters: never edit `site/` by hand

`site/` is **build output**. It is regenerated from `src/` every time and is
git-ignored. Editing a file in `site/` will be silently overwritten on the next build.

**To change the site, edit the generator in `src/`, then run `python3 build.py`.**

  assets/          logos, icons, and photos/ (the real imagery)

  *_test.js        interaction tests (mega menu, review rotation, video facades)

worker/
  src/index.js         routes, lead/application/data-request handlers, admin API

## Common tasks

- **Build**: `python3 build.py` (or `npm run build`)
- **Preview**: `npm run serve` → http://localhost:8000
- **Test**: `npm test` (site interaction tests + worker/auth tests)
- **Deploy site**: `./deploy.sh`
- **Deploy backend**: `cd worker && ./setup.sh`

## Hard-won lessons (please respect these)

- **`str.replace()` that silently matches nothing** has bitten this project repeatedly.
  When you replace an anchor string in a generator, `assert` it was present first, or
  the change vanishes with no error.
- **CSS specificity leaks**: `nav.links a` once bled into `.mega-card` and broke the
  menu twice. Nav styles are scoped to direct children (`nav.links > a`). After any CSS
  change touching the nav or mega menu, run `python3 src/cascade_check.py` (build.py
  does this automatically).
- **Nested `assets/`**: `cp -r assets site/assets` into an existing folder creates
  `assets/assets/` and doubles the payload. build.py uses `dirs_exist_ok` correctly;
  don't reintroduce the manual copy. validate.py fails the build if it reappears.
- **Don't reason about the CSS cascade in your head** — run cascade_check.py.
- **JS that targets renamed elements dies silently**. The review rotation and mega menu
  both broke this way (querySelector → null → early return, no error). The `*_test.js`
  files actually execute the JS against a mock DOM; run them after touching interactions.
- **Titles must stay ≤ 60 chars** or Google truncates them. validate.py enforces this.
- **The homepage must not repeat a content photo** in its body (mega menus excepted).
  validate.py enforces this too.
- **Videos are click-to-load facades**, never raw iframes — keeps YouTube (and its
  cookies) from loading until the user asks. Don't add a bare `<iframe>`.

## Deploy facts

- Cloudflare Pages project name: `prohealth` (subdomain `prohealth-1oi.pages.dev`)
- Worker: `prohealth-api.calm-hall-dd9a.workers.dev`, admin at `/admin`
- `setup.sh` caches resource IDs in `~/.prohealth-ids` so a fresh clone restores them
- After a fresh clone, run `cd worker && ./setup.sh` — never `wrangler deploy` directly
  (wrangler.toml ships with placeholder IDs that setup.sh fills in)

## Before going live on prohealth.us

Read `PRE-LAUNCH.md`. The real risk is email: moving nameservers to Cloudflare makes it
authoritative for all DNS, so MX/SPF/DKIM must be recreated or mail breaks silently.
Also unresolved and flagged there: the NAP address conflict, unverified review counts,
and compliance pages that no lawyer has reviewed (`LEGAL-REVIEW.md`).
